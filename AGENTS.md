# Repository Instructions

## Documentation Policy

Any change that modifies files in this repository must also update `DOCS.md` when needed so it continues to reflect the current repository structure, behavior, and usage.

This applies to:

- creating files
- deleting files
- renaming files or directories
- changing module responsibilities
- changing setup or usage flows
- adding or removing important dependencies

If a code or configuration change does not affect repository structure, behavior, or usage, `DOCS.md` does not need to be edited just for the sake of touching it.
